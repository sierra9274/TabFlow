How It Works

1

Triggers Monitor

The extension runs in a background service worker, listening to changes (creating a tab, updating an address, closing a tab). It calculates tab counts independently for each browser window.

2

Classification Logic

When the threshold is breached, TabFlow runs every URL, host domain, and page title through the classification rules. Custom rules you create take priority over default settings.

3

Chrome Tab Groups API

Tabs are consolidated using Chrome's native chrome.tabs.group engine. TabFlow names folders, assigns HSL matched colors, and collapses folders to maximize horizontal tab space.

Technical Resume Highlights

This project showcases the following engineering skills:

  • Chrome Extensions MV3 Architecture: Background service worker lifecycles, asynchronous event listener optimizations, storage synchronization.
  • Asynchronous Scripting & Debouncing: Implemented precise event-debouncing in the background worker to avoid overlapping API transactions and rate-limiting during high tab-creation bursts.
  • Dynamic SVG/CSS Theming: Responsive dark mode dashboard styled with modular glassmorphic principles, standard CSS Custom Properties (Variables), and dynamic transitions.
  • State Serialization: Syncing user categorization tables and custom settings using Chrome's asynchronous Key-Value storage (chrome.storage.local).
